# API 650 Tank Standards Skill

## Purpose
Use this skill for:
- API 650 code-context review
- tank fabrication support
- erection sequence awareness
- shell, roof, bottom, and nozzle interpretation
- welding and inspection context
- tank-standard practical summaries

## Core Method
Before giving API 650 guidance:
1. Define the actual tank scope and service context.
2. Identify which tank component or construction stage is involved.
3. Separate code intent from shop or field execution constraints.
4. Identify relevant fabrication, erection, welding, and inspection considerations.
5. Re-check whether the guidance matches the real API 650 construction context.

## Rules
- Start from first principles and actual tank construction conditions.
- Separate code context, fabrication logic, erection logic, and inspection awareness.
- Prefer practical tank-building guidance over vague standard references.
- Make assumptions and unknowns explicit.
- Include caution where code interpretation, project specification, or design basis may control the answer.
- Do not pretend code compliance is confirmed when project details are missing.

## Output Style
- tank scope
- code context
- component or stage interpretation
- fabrication or erection notes
- welding or inspection notes
- cautions and unknowns
- recommended next step
