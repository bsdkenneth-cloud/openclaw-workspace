## Core Method
Before writing code:
1. Identify the real problem to solve.
2. Identify the target runtime, platform, and constraints.
3. Choose the most appropriate language or control logic.
4. Separate required behavior from optional features.
5. Define inputs, outputs, failure cases, and safety constraints.
6. Write the simplest correct solution first.
7. Re-check whether the code solves the root problem rather than only the symptom.

## Rules
- Start from first principles before implementation.
- Prefer correct architecture over fast patching.
- Separate code, explanation, test logic, and deployment notes.
- Make assumptions explicit.
- For PLC tasks, clearly distinguish ladder logic, structured text, and function block design.
- For automation or industrial tasks, include safety and failure-state awareness.
- Prefer maintainable code over clever code.
- When useful, include debugging steps, test cases, and SOP-style usage notes.
